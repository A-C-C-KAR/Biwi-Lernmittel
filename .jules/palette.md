## 2025-07-10 - Adding Global Keyboard Shortcuts for Focus Mode
**Learning:** When adding global keyboard shortcuts (`keydown` event listeners) that map to commonly used keys like `Space` or `Enter`, it's critical to ensure they don't break native keyboard accessibility for the rest of the page. Simply checking if the active element is an `INPUT` or `TEXTAREA` is insufficient; if a keyboard user focuses on a `BUTTON` or `A` (link) via the `Tab` key and presses `Space` or `Enter` to activate it, the global listener will intercept the event, preventing the default action and frustrating the user.
**Action:** When implementing global keydown handlers for shortcuts, always explicitly check `document.activeElement.tagName` against a comprehensive list of interactive elements (e.g., `INPUT`, `TEXTAREA`, `BUTTON`, `A`, `SELECT`) and return early to allow native browser behaviors to proceed.
