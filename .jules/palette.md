## 2024-07-22 - Converting Custom Accordions to Native Buttons
**Learning:** Custom `<div>` elements used as interactive accordions (with `onclick`) inherently lack keyboard accessibility (they can't receive focus natively via Tab) and semantics for screen readers. In vanilla JavaScript applications without frameworks to handle this automatically, replacing the `<div>` with a semantic `<button>` tag and manually managing the `aria-expanded` state on click is crucial for accessibility.
**Action:** When implementing or modifying accordion components in vanilla JS, always use `<button>` for the trigger, apply `focus:outline-none focus-visible:ring-2` for visual keyboard focus, and programmatically toggle `aria-expanded` when the visibility of the controlled content changes.
