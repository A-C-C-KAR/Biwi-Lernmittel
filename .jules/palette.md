## 2025-01-28 - Accessible Accordions in Vanilla JS
**Learning:** In vanilla JS applications lacking a component library, interactive elements like accordions often incorrectly use generic `<div>` tags with `onclick` handlers, rendering them inaccessible to keyboard users and screen readers. Managing ARIA states (like `aria-expanded` and `aria-controls`) and focus states manually is crucial for these components.
**Action:** When adding or modifying interactive UI components in vanilla JS, always prioritize semantic HTML (e.g., `<button>` for clickable headers) and ensure `aria-expanded` states are properly synchronized via JS when the component's visibility changes. Use existing CSS utilities (like Tailwind's `focus-visible:ring-2`) to provide clear visual feedback for keyboard navigation.
